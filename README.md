# Панель агронома

Готовый стартовый проект под **картофельный производственный цикл**: список полей, лента операций по полю, аналитика, сравнение полей, справочники сортов и удобрений. Проект сделан **без Supabase и без внешней BaaS-платформы** — данные хранятся в **PostgreSQL**, доступ идёт через `pg` и SQL.

## Что уже сделано

- Главный экран со списком полей
- Детальная страница поля в формате чат-ленты
- Сводные KPI по хозяйству
- Графики по болезням и связи калия с урожайностью
- Таблица сравнения полей
- JSON API для полей, аналитики и справочников
- API для создания новой операции
- SQL-схема и seed-данные
- Docker Compose для локального Postgres
- Нормальный UI на Next.js + Tailwind

## Стек

- Next.js 14
- React 18
- TypeScript
- Tailwind CSS
- PostgreSQL 16
- node-postgres (`pg`)
- Recharts

## Запуск

### 1. Поднять Postgres

```bash
docker compose up -d
```

### 2. Создать `.env`

```bash
cp .env.example .env
```

### 3. Установить зависимости

```bash
npm install
```

### 4. Инициализировать БД

```bash
npm run db:seed
```

### 5. Запустить проект

```bash
npm run dev
```

После этого приложение будет доступно на `http://localhost:3000`.

## Структура

```text
app/
  api/

  init.sql
  seed.sql
scripts/
  seed.js
docker-compose.yml
```

## Основные маршруты

- `/` — дашборд и список полей
- `/fields/1` — карточка поля и таймлайн операций
- `/api/fields` — список полей
- `/api/fields/:id` — детали поля
- `/api/analytics/summary` — аналитика для главного экрана
- `/api/references/varieties` — справочник сортов
- `/api/references/fertilizers` — справочник удобрений
- `/api/operations` — создание новой записи по полю

## Пример POST на создание операции

```bash
curl -X POST http://localhost:3000/api/operations \
  -H "Content-Type: application/json" \
  -d '{
    "field_id": 2,
    "operation_type": "inspection",
    "operation_date": "2026-06-15",
    "title": "Повторный осмотр",
    "notes": "Риск фитофтороза снизился после фунгицида",
    "payload": {
      "emergencePct": 89,
      "plantDensity": 42,
      "stemsPerPlant": 3.9,
      "lateBlight": 2,
      "alternaria": 1
    },
    "current_phase": "клубнеобразование",
    "disease_status": 2
  }'
```

## Важно

Сейчас это **готовая рабочая база проекта**, которую можно сразу положить в репозиторий и развивать. Я специально не завязывал архитектуру на сторонние backend-платформы: только приложение + PostgreSQL + SQL.
